# Integrated Child Profile Architecture (Read-Only Foundation)

## Bounded Contexts
- The Gates remains the rite-of-passage developmental maturity system.
- Youth Development/TDE remains the longitudinal signal/intervention system.
- Archetype/Identity remains the tendency/archetype interpretation system.

## Separation Principle
No scoring systems are merged. No source schema ownership is changed. The bridge composes read-model output only.

## Bridge Philosophy
The bridge uses soft, non-diagnostic language:
- emerging tendencies
- developmental supports
- stabilizing practices

It avoids fixed labels and deterministic identity claims.

## Progressive Disclosure UX
Integrated payload is additive; existing experiences remain sovereign. UI can progressively reveal integrated sections only when multi-source context exists.

## Safeguards
- Non-diagnostic framing preserved.
- No writes from integrated profile load.
- No scoring mutations.
- Graceful degradation when one or more systems are missing.

## Future Direction
A future habit-bank layer can map observed supports and integration signals without turning practices into compliance scoring.
